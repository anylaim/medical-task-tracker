# Task Service

Сервис для управления задачами с HTTP API на Go.

## Требования

- Go `1.23+`
- Docker Рё Docker Compose

## Быстрый запуск через Docker Compose

```bash
docker compose up --build
```

После запуска сервис будет доступен по адресу `http://localhost:9000`.

Если `postgres` уже запускался ранее со старой схемой, пересоздай volume:

```bash
docker compose down -v
docker compose up --build
```

Причина в том, что SQL-файл из `migrations/0001_create_tasks.up.sql` монтируется в `docker-entrypoint-initdb.d` и применяется только при инициализации пустого data volume.

## Swagger

Swagger UI:

```text
http://localhost:9000/swagger/
```

OpenAPI JSON:

```text
http://localhost:9000/swagger/openapi.json
```

## API

Базовый префикс API:

```text
/api/v1
```

Основные маршруты:

- `POST /api/v1/tasks`
- `GET /api/v1/tasks`
- `GET /api/v1/tasks/{id}`
- `PUT /api/v1/tasks/{id}`
- `DELETE /api/v1/tasks/{id}`

## Реализация периодичности (Тестовое задание)

В рамках задания была добавлена возможность создания периодических задач. 

### Основные изменения:
1.  **Модель данных**: В структуру `Task` добавлены поля для настройки периодичности: `recurrence_type`, `recurrence_value`, `specific_dates` и `parity_type`.
2.  **Бизнес-логика (Usecase)**: При создании задачи с типом `daily`, система автоматически генерирует серию из 10 задач вперед с интервалом, указанным в `recurrence_value`. Это позволяет врачам сразу видеть план процедур в трекере.
3.  **Гибкость**: Архитектура подготовлена для реализации других типов периодичности (ежемесячно, четные/нечетные дни), так как соответствующие поля уже поддерживаются API и транспортным слоем.

### Технические решения и предположения:
- **Автоматическая генерация**: На данный момент при выборе ежедневного типа создается фиксированная серия (10 задач). В реальной системе это число можно вынести в настройки или сделать бесконечным через фоновые воркеры.
- **Порты**: Для удобства разработки в Docker-окружении основной порт изменен на `9000` (проброс с внутреннего `8080`).
- **Кодировка**: Обеспечена полная поддержка UTF-8 для корректной работы с кириллицей в названиях задач и описаниях (исправлены проблемы отображения в терминале и БД).

### Пример запроса для создания серии задач:
```json
POST /api/v1/tasks
{
  "title": "Ежедневный обход",
  "description": "Проверка состояния пациентов в отделении",
  "status": "new",
  "recurrence_type": "daily",
  "recurrence_value": 1
}
